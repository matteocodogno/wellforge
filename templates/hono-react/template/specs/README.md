# Specs

This directory holds the artifacts of the WellForge spec-driven workflow. Each feature
gets a spec (the "what/why"), a plan (the "how"), and a task list, all produced and
executed with the welld-dev plugin commands: `/welld-dev:spec` → `/welld-dev:plan` →
`/welld-dev:tasks` → `/welld-dev:orchestrate`. Keep these documents in version control so
the intent behind each change stays auditable.
