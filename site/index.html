Hi, I'm Forgey — your AI blacksmith. I forge the boring part (scaffold, gates, CI, the spec workflow, releases) so you get straight to building what actually matters.

The problem

Every new project pays the same tax

Wire up the AI agents, the spec workflow, the scaffold, the quality gates, the CI, the tool connections. Hours of it, every time. Everyone does it a little differently — and it quietly rots the moment your standards move on.

What WellForge is

Six pillars, one command

One interview, and you get a building, CI-gated, spec-driven, AI-ready repository — with a team of AI agents already wired in.

Pillar 1

Spec-driven framework

A standardized spec → plan → tasks workflow, owned in-house so it encodes your conventions and doesn't churn under you.

Pillar 2

Multi-agent team

Product Owner, Architect, Designer, Frontend & Backend Devs, DevOps, Quality Engineer — each a Claude Code subagent with a crisp role.

Pillar 3

AI orchestrator

Routes work to the right agents, enforces the spec workflow, and coordinates handoffs from PO → Architect → Devs → QE.

Pillar 4

Scaffolder + connections

Recommends a stack, generates from a versioned template, and guides MCP/CLI connections to GitHub, CI, cloud, and observability.

Pillar 5

Quality gates as law

Coverage, lint, types, SAST, dependency audits — defined centrally, enforced in CI and locally. Never left to individual discretion.

Pillar 6

Upgradeable lifecycle

Every project records the template + version that made it, and can be re-templated when presets evolve. A scaffold that doesn't rot.

How it fits together

Three layers, because a plugin alone can't

Pillars 1–3 live in the AI session; pillars 4–6 need artifacts that outlive it — versioned templates and CI workflows. So WellForge is three cooperating layers.

wellforge-plugin/

Claude Code plugin

Skills, agents, commands, hooks, MCP — the spec workflow, the agent team, the orchestrator, and local gate enforcement.

Covers: pillars 1, 2, 3
copier.yml + templates/

Copier template

A monorepo template chosen for first-class copier update re-templating — the scaffolder and the upgradeable lifecycle.

Covers: pillars 4, 6
.github/workflows + gates/

Reusable CI gates

Central thresholds and SAST configs, referenced (not copied) by every scaffold, so raising the bar is one PR for the whole fleet.

Covers: pillar 5
Match ceremony to stakes

Rigor tiers

A throwaway prototype and a regulated production service shouldn't pay the same setup cost. Pick a tier per feature — a security floor holds in every one.

spike

The main-loop fast lane: build from a one-paragraph brief, advisory gates, no agents, no approval. Feasibility work in minutes.

mvp

Coverage advisory, SAST still blocks. Enough rigor to validate, tracked as debt you can pay down later.

production

Full gates — 80% coverage, SAST, LM-judge eval. Graduate up with /wellforge:promote; production only on an eval pass.

Get started

From nothing to building

Works with Claude Code and OpenCode.

1

Install the plugin

Add the WellForge marketplace and install the plugin — the agents, commands, and skills come with it.

2

Run the front door

/wellforge:new interviews you, recommends a stack, generates the scaffold, verifies the build, and walks the connections.

3

Or adopt an existing repo

/wellforge:adopt brings the workflow and calibrated gates to a brownfield project — one layer at a time, additive only.

4

Build with the team

/wellforge:orchestrate drives the full agent pipeline on a goal: spec → plan → tasks → code → QE → eval.

# install (local marketplace)
/plugin marketplace add matteocodogno/wellforge
/plugin install wellforge

# scaffold a new project…
/wellforge:new

# …or adopt an existing one
/wellforge:adopt
